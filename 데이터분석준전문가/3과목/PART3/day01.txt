3-79. 데이터 마이닝 개요
데이터 마이닝 
    기업이 보유하고 있는 일일 거래 데이터, 고객 데이터, 상품 데이터 혹은 각종 마케팅 활동에
    있어서의 고객반응 데이터 등과 이외의 외부 데이터를 포함하는 모든 사용가능한 원천 데이터를
    기반으로 감춰진 지식, 기대하지 못했던 경향 또는 새로운 규칙 등을 발견하고 이를 실제 비즈니스
    의사결정 등에 유용한 정보로 활용하는 일련의 작업


데이터 마이닝 5단계
    1단계. 목적 정의
        - 데이터 마이닝 도입 목적을 명확하게 한다.
    2단계. 데이터 준비
        - 데이터 정제(Cleaning)를 통해 데이터의 품질 확보를 한다.
        - 필요 시 데이터양을 충분하게 확보한다.
    3단계. 데이터 가공
        - 목적 변수를 정의하고, 필요한 데이터를 데이터 마이닝 소프트웨어에 적용할 수 있게
          가공 및 준비하는 단계
        - 충분한 CPU와 메모리, 디스크 공간 등 개발환경 구축을 선행한다.
    4단계. 데이터 마이닝 기법 적용
        - 모델을 목적에 맞게 선택하고 소프트웨어를 사용하는 데에 필요한 값을 지정한다.
    5단계. 검증
        - 결과에 대한 검증을 시행한다.


대표적인 데이터 마이닝 기법
    ✔. 분류(Classification)
        - 새롭게 나타난 현상을 검토하여 '기존의 분류, 정의된 집합에 배정'하는 것이다.
        - 예: 의사결정나무, memory-based reasoning 
    ✔. 추정(Estimation)
        - 주어진 입력 데이터를 사용하여 '알려지지 않은 결과의 값을 추정'하는 것이다.
        - 연속된 변수의 값을 추정한다. 
        - 예: 신경망 모형
    ✔. 연관분석(Association Analysis)
        - '같이 팔리는 물건' 같이 아이템의 연관성을 파악하는 분석
        - 예: 카탈로그 배열 및 교차판매, 공격적 판촉행상 등의 마케팅 계획
    ✔. 예측(Prediction)
        - 미래에 대한 것을 예측, 추정하는 것을 제외하면 분류나 추정과 동일한 의미 
        - 예: 장바구니 분석, 의사결정나무, 신경망 모형
    ✔. 군집(Clustering)
        - 미리 정의된 기준이나 예시에 의해서가 아닌 '레코드 자체'가 가진 다른 레코드와의
          유사성에 의해 그룹화되고 이질성에 의해 세분화된다.
        - 데이터 마이닝이나 모델링의 준비단계로써 사용된다.
    ✔. 기술(Description), 명세, 설명하다의 '기술'
        - 데이터가 가진 특징 및 의미를 단순하게 설명하는 것이다.
        - 데이터가 암시하는 바에 대해 설명 및 그에 대한 답을 찾아낼 수 있어야 한다.


*전처리(PreProcessing)
    - 원자료(Raw Data)를 데이터 분석 목적과 방법에 맞는 형태로 처리하기 위하여 불필요한 정보를 
      분리 제거하고 가공하기 위한 예비적인 조작
    
    전처리 방법의 예
        ✔. 데이터 정제
            결측 값(Missing Value)들을 채워넣고, 이상치를 식별 또는 제거하고 잡음 섞인 데이터를
            평활화(Smoothing)하여 데이터의 불일치성을 교정하는 기술
        ✔. 데이터 변환
            데이터 유형 변환 등 데이터 분석이 쉬운 형태로 변환하는 기술.
            정규화, 집합화, 요약, 계층 생성 등의 방법을 활용한다.
        ✔. 데이터 필터링
            오류 발견, 보정, 삭제 및 중복성 확인 등의 과정을 통해 데이터의 품질을 향상하는 기술
        ✔. 데이터 통합
            데이터 분석이 용이하도록 유사 데이터 및 연계가 필요한 데이터들을 통합하는 기술
        ✔. 데이터 축소
            분석 시간을 단축할 수 있도록 데이터 분석에 활용되지 않는 항목 등을 제거하는 기술

로지스틱 회귀분석 
    - 독립변수는 연속형, 종속변수가 범주형인 경우 적용되는 회귀분석 모형
    - 종속변수가 성공/실패, 사망/생존과 같이 이항변수(0, 1)로 되어있을 때 종속변수와 독립변수
      간의 관계식을 이용하여 두 집단 또는 그 이상의 집단을 분류하고자 할 때 사용되는 분석기법

선형회귀와 로지스틱회귀의 비교
    ✔.선형회귀 
        - x값의 변화에 따른 y값의 변화를 알아낸다.
        - x가 1 증가할 때 y는 회귀계수만큼 증가한다.
    ✔.로지스틱회귀 
        - x값에 따른 y값의 변화량의 문제가 아니다.   ? 
        - 회귀계수를 해석할 때 문제가 생긴다.        ? 

로지스틱 회귀분석
    종속변수를 전체 실수 범위로 확장하여 분석하고 sigmoid 함수를 사용해 연속형 0~1값으로 변경

    ＠순서
    ＠회귀식에 대한 해석 방법이 선형 회귀와 다르다.
        01. Probability(확률, 개연성)
            - 0 ~ 1 사이의 값
        02. Odds(승산)
            - 성공률/실패율, P₁/(1-P₁), P₁ = 성공률 
                - 성공이 일어날 가능성이 높은 경우는 1.0보다 큰 값
                - 실패가 발생할 가능성이 높은 경우는 1.0보다 작은 값
            - 로지스틱의 회귀계수, 확률에 대해 0 ~ ∞로 변환한 값
        03. Log Odds(=Logit)
            - log(odds), odds값에 log를 취한다.
            - 선형화의 하나, 값의 범위를 전체 실수 범위(-∞ ~ +∞)로 확장한다.
            - log(odds(p)) = wx + b 형태로 선형분석이 가능해진다.
        04. sigmoid(구불한 모양, S자 형태)
            - log odds값을 연속형 0~1 사이의 값으로 바꾸는 함수
            - Logistic 함수라 불리기도 하며 비선형적 값을 얻기 위해 사용한다.
