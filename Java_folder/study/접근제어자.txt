접근제어자 : 외부에서 접근하지 못하도록 제한하는 역할을 한다.


    private : 같은 클래스 내에서만 접근이 가능하다.
    protected : 같은 패키지 내에서도 접근이 가능하며 다른 패키지의 자식 클래스에서는 접근이 가능하다.
    public : 접근 제한이 없다.
    dafault : 같은 패키지 내에서만 접근이 가능하다(디렉터리).
              보통 아무런 표시가 없다면 default이다.

    * public > protected > default > private 

접근 제어자를 사용하는 이유 : 클래스 내부에 선언된 데이터를 보호하기 위함이다.
데이터가 유효한 값을 유지할 수 있도록 또는 비밀번호와 같은 중요한 데이터를 외부에서 함부로 변경하지 못하게 외부로부터의 접근을 제한하는 것이 필요하다. 외부에서 접근할 필요가 없는 멤버 변수들은 Private으로 지정하여 외부에 노출을 막음으로 복잡성을 줄일 수 있다(=캡슐화).
