사용자 차트(USER FLOW) 혹 이용 흐름 설계란?
    사용자와 프로덕트가 어떻게 상호작용하는지 시각화해 보여주는 차트다.
    Task Flow, Wire Flow, Flow Chart 등 비슷한 용어가 많다.

    정의
        - 사용자가 특정한 작업을 완료하기 위해 서비스 내에서 움직이는 경로는 정리한다.
        - 사용자 시나리오보다 구체적이며 UI(화면), 행동, 판단으로 표현한다.

    구성요소
        - 목표: 사용자가 서비스를 통해 얻고자 하는 결과 예) 상품 구매, 정보 획득 등 
        - 작업: 목표를 위해 유저가 수행하는 것 예) 로그인, 회원가입, 장바구니, 검색 등
        - 화면: 예) 검색화면, 필터링, 결제창 비밀번호 입력 팝업 등
        - 행동: UI 상의 주 활동 및 보조 활동 예) 필터링, 선택, 예약, ID 혹 PW 입력 
        - 판단: 사용자 행동(로직) 후 그 결과가 분기될 경우 예) 로그인 되었는가? 결과값을 찾았는가?

    고려요소 
        · 사용자 흐름은 최대한 간단하게 만드는 것이 핵심이다.
        · 사용자들이 쉽게 접근하고 사용할 수 있도록 설계한다.
        · 로그인 창과 회원가입 창은 이해하기 쉽게 만든다.
