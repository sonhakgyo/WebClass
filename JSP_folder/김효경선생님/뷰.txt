커스텀 태그
        · 화면의 코드를 단순하게 변경 가능하다.
        · 로직 등을 화면에서 숨길 수 있다(=캡슐화). 
        → 보다 완벽한 MVC 패턴 

    1) 태그파일 기반
            - 비교적 구현이 간단하다.
    2) 태그 핸들러클래스기반(.java)
            - 배포가 용이하다.
            - 커스텀 태그 정의 파일이 필요하다.

            XML 파일: 설정 파일을 의미한다. JSP보다는 SPRING에서 많이 보인다.
