MVC 패턴
        M : Model
        V : View 
        C : Controller

Model 
        데이터베이스와의 관계를  담당, 클라이언트의 요청에 의해서 필요한 자료를 데이터베이스로부터 추출하거나
        수정해서 컨트롤러로 전달하는 역할이다.
    
View 
        사용자에게 보여지는 UI 화면, 주로 .jsp 파일로 작성하며 컨트롤러에서 어떤 뷰 컴포넌트를 보여줄 지 결정한다.
    
Controller
        클라이언트의 요청을 받고 적절한 모델에 지시를 내리며 모델에서 전달된 데이터를 적절한 뷰에 전달하는 역할이다.


진행
        클라이언트 -> 컨트롤러 -> 모델 -> 데이터베이스 -> 모델 -> 컨트롤러 -> 뷰 -> 컨트롤러

모델1 : 뷰와 컨트롤러가 같이 있는 형태
        클라이언트 -> JSP(뷰+컨트롤러) <-- --> DAO(모델) <-- --> 데이터베이스
        ** 모델1은 개발기간이 짧은 소규모 프로젝트, 변경사항이나 요구사항이 적을 경우 빠르게 개발할 수 있다는 장점이 있다.
           하지만 재활용성(재사용성)이 떨어진다는 단점이 있다.

모델2 : MVC에서 모델, 뷰, 컨트롤러 모두 모듈화 되어있는 형태
        ** 개발 초기에는 부하가 비교적 높고, 구조(아키텍쳐)에 대한 수준 높은 이해가 필요(요구)하다.
           재사용성이 높고 확장성이 용이하다.
